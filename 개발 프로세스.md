## 개발 프로세스 

* 개발을 어떻게 어떠한 절차를 거쳐서 해야하는지?

* 일하는 방식, 여러사람이 협업하면서 어떻게 하는지 

  * 1.  요구사항 needs 파악 : vision, 요구사항 정의서
    2. 업무를 이해 분석하고 : 인프라(서버, 네트워크 등), 어플리케이션의 구성
    3. 시스템 기반을 설계하고 : 클라이언트/서버,  인터페이스
    4. 세세한 부분을 설계 : 프론트/백엔드, 저장소,
    5. 개발 테스트: 테스트
    6. 운영환경으로 이관하고 운영한다.
